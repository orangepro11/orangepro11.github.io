在vue3中使用composition api代替vuex

orangepro · 2021-9-20 · 本文共 次阅读 · 字数统计: 1.1k字 · 阅读时长≈ 4分钟 ·

温馨提示

本demo仅作为一次尝试，不要在实际的工程项目中使用本套方案作为vuex的替代方案！

使用vue-cli创建新的vue3项目

  1. 首先安装vue-cli
    npm install -g @vue/cli
  2. 引导初始化
    vue create composition-api-state-management
  3. 勾选vue-router,取消ESLint
  4. 等待初始化完成

经典的计数器

本次演示的demo基本都会写在这个文件里，里面的初始代码都可以删掉，只有少数部分会写在App.vue里

<template>
  <div class="home">
    <div class="counter">{{ counter }}</div>
    <div class="buttons">
      <button @click="decreaseCounter">-</button>
      <button @click="increaseCounter">+</button>
    </div>
  </div>
</template>

这样就可以了，作为演示用不需要做的太复杂，然后就是script部分。

<script>
import { ref } from 'vue';

export default {
  setup() {
    let counter = ref(0);

    const increaseCounter = () => {
      counter.value++;
    }

    const decreaseCounter = () => {
      counter.value--;
    }

    return {
      counter,
      increaseCounter,
      decreaseCounter
    }

  }
}
</script>

以下是可选的样式，不加也不影响正常功能：

<style>
div {
  margin-bottom: 10px;
}

.counter {
  font-size: 80px;
}
.buttons button {
  font-size: 40px;
  width: 100px;
  margin: 0 10px;
}
</style>

语法上使用vue3的composition api，ref创建一个响应式的基本数据类型（不要滥用reactivie），setup()即为页面应用的启动入口，同时定义了两个方法分别为计数器增和计数器减，通过return的方式把这些暴露在上下文中供模板去使用。

现在我们的计数器已经可以正常工作了，cool~~，但是不对，这不是我们的目的，我们要改成用状态管理器的东西（但不用vuex）。

状态管理

新建个store目录，index.js

import { reactive } from 'vue';

const state = reactive({
    counter: 0,
});

export default {
    state,
}

这样我们就创建了一个状态集合，目前只有一个counter，现在怎么让它可以在我们的应用中被访问到呢？

Provide和Inject

字面意思，提供和注入：在父组件中提供在子组件中注入。为了方便直接在根组件中注入。

App.vue

<script>
import store from '@/store';
import { provide } from 'vue';

export default {
  setup() {
    provide('store', store);
  }
}
</script>

@的意思就是项目的根目录，这个映射已经自动做好了，只管用就好了。
通过provide api去提供数据，第一个参数就是key。

Home.vue


<script>
import { ref, inject } from 'vue';

export default {
  setup() {
    const store = inject('store');

    let counter = ref(0);

    const increaseCounter = () => {
      counter.value++;
    }

    const decreaseCounter = () => {
      counter.value--;
    }

    return {
      store,
      counter,
      increaseCounter,
      decreaseCounter
    }

  }
}
</script>

通过inject api完成注入

然后在模板中使用store里的状态：

<template>
  <div class="home">
    <div class="counter">{{ store.state.counter }}</div>
    <div class="buttons">
      <button @click="decreaseCounter">-</button>
      <button @click="increaseCounter">+</button>
    </div>
  </div>
</template>

mutations

现在状态有了，虽然我们可以直接修改它的值，但这显然不符合“状态管理”的哲学。我们需要一个专门去修改状态的方法并进行统一的管理。

store/index.js

import {
    reactive
} from 'vue';

const state = reactive({
    counter: 0,
});

const methods = {
    decreaseCounter: () => {
        state.counter--;
    },

    increaseCounter: () => {
        state.counter++;
    }
}

export default {
    state,
    methods,
}

现在我们可以替代Home.vue中的计数器增和计数器减的方法了。

Home.vue

<template>
  <div class="home">
    <div class="counter">{{ store.state.counter }}</div>
    <div class="buttons">
      <button @click="store.methods.decreaseCounter">-</button>
      <button @click="store.methods.increaseCounter">+</button>
    </div>
  </div>
</template>

我们同时可以检测它在其他页面中是否也可以正常工作：

About.vue

<template>
  <div class="about">
    <h1>{{ store.state.counter }}</h1> 
  </div>
</template>

<script>
  import { inject } from 'vue';
  export default {
    setup() {
      const store = inject('store');

      return {
        store,
      }
    }
  }
</script>

优化部分

专门的getter

store/index.js

import {
    reactive
} from 'vue';

const state = reactive({
    counter: 0,
});

const methods = {
    decreaseCounter: () => {
        state.counter--;
    },

    increaseCounter: () => {
        state.counter++;
    }
}

const getters = {
    counter: () => state.counter,

    counterSquared: () => state.counter ** 2,
}

export default {
    state,
    methods,
    getters,
}

如此我们就可以通过getter访问这个计算属性：

{{ store.getters.counterSquared() }}

状态是只读的

我们一般不希望state里的数据被随意的改动，我们希望所有的改动都是可控的，不容忍任何预期之外的改动，所以类似于使用v-model去直接改变state里的数据是不被允许的

store/index.js

import {
    reactive, readonly
} from 'vue';

const state = reactive({
    counter: 0,
});

const methods = {
    decreaseCounter: () => {
        state.counter--;
    },

    increaseCounter: () => {
        state.counter++;
    }
}

const getters = {
    counter: () => state.counter,

    counterSquared: () => state.counter ** 2,
}

export default {
    state: readonly(state),
    methods,
    getters,
}

通过readonly修饰state，和const不同，它会深度劫持这个对象，通过defineProperty的方式确保对象里的每一个属性都是不可写入的，这样被导出的state就是不可更改的，只能通过methods去更改它。
