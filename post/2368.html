JavaScript中的深拷贝和浅拷贝

orangepro · 2021-1-23 · 本文共 次阅读 · 字数统计: 1.1k字 · 阅读时长≈ 4分钟 ·

前言

首先希望你认真阅读这部分内容，这对你理解什么是浅拷贝和深拷贝非常的有帮助，不至于产生困惑。

我们将通过JavaScript语法构建一个对象名为student，它有name和score两个属性，而score又会包含若干属性表示成绩，在之后的例子中他们将反复出现。

let studentA = {
    name:'A',
    score:{
        Math:43,
        English:80
    }
}

以上代码表示一个同学名为A，数学43分，英语80分，它将不再重复出现。

现在请注意分析以下代码：

实例一

let studentB = studentA
studentB.name = 'B'

console.log(studentA)

以上代码的运行结果为

{ name: 'B', score: { Math: 43, English: 80 } }

相信大家对这个结果是没有异议的，因为studentA和studentB实际上指向了一个内存地址，所以任何的更改必然同时影响A和B。现在的问题是：这个叫什么？

回答“赋值”的人，恭喜你回答正确；对于回答“浅拷贝”的人，请继续往下看

实例二

let studentB = JSON.parse(JSON.stringify(studentA))
studentB.name = 'B'

console.log(studentA)

以上代码的运行结果为

{ name: 'A', score: { Math: 43, English: 80 } }

这个也很好理解，经过JSON序列化后他们显然不再有直接的关系了，那么，这个叫什么？

没错，就是深拷贝，现在请继续往下看，

实例三

let studentB = Object.assign({},studentA)
studentB.name = 'B'

console.log(studentA)

以上代码的运行结果为

{ name: 'A', score: { Math: 43, English: 80 } }

对于不熟悉Object.assign()方法的，请参考这个链接，现在要来问，这个叫什么？

首先要恭喜回答浅拷贝的人回答正确，如果你的回答是深拷贝，请思考如下代码的运行结果

studentB.score.Math = 98

console.log(studentA)

运行结果是

{ name: 'A', score: { Math: 98, English: 80 } }

发现问题了吗？studentA里的name虽然没变，但是score却受了影响，换句话说，它可以复制一份值对象，但是对于引用对象却无能为力。这就叫浅拷贝

正文

前面我们分别讲述了赋值，深拷贝，浅拷贝这三种情况，这里我就不对深拷贝和浅拷贝下定义了，因为一定会把你们绕迷糊的，所以我们只讲述深浅拷贝的若干种方法的特性和优缺点。

使用ES6的扩展运算符实现浅拷贝

这个方法实际上和object.assign()方法有异曲同工之妙，写起来更简单一些，但是我还是要说，它不能替代Object.assgin()，具体体现在对象的合并上，这里我就不展开叙述，不然就偏题了。

let studentB = {...studentA}

借助JSON的工具方法实现深拷贝

let studentB = JSON.parse(JSON.stringify(studentA))

以上方法可以满足大部分的深拷贝需求，但是请注意：此方法对undefined,RegExp,function等类型无效，如果你的对象中含有这些类型，请注意他们会被丢失。

使用Object.create()方法完成深拷贝

function deepCopy(obj) {
  var copy = Object.create(Object.getPrototypeOf(obj));
  var propNames = Object.getOwnPropertyNames(obj);

  propNames.forEach(function(name) {
    var desc = Object.getOwnPropertyDescriptor(obj, name);
    Object.defineProperty(copy, name, desc);
  });

  return copy;
}

let studentB = deepCopy(A)

使用递归算法完成深拷贝(无坑版)

function isPrimitive(value){
  return (typeof value === 'string' || 
  typeof value === 'number' || 
  typeof value === 'symbol' ||
  typeof value === 'boolean')
}

/**
 * 判断是否是一个js对象
 * @param value 
 */
function isObject(value){
  return Object.prototype.toString.call(value) === "[object Object]"
}

/**
 * 深拷贝一个值
 * @param value 
 */
function cloneDeep(value){

  // 记录被拷贝的值，避免循环引用的出现
  let memo = {};

  function baseClone(value){
    let res;
    // 如果是基本数据类型，则直接返回
    if(isPrimitive(value)){
      return value;
    // 如果是引用数据类型，我们浅拷贝一个新值来代替原来的值
    }else if(Array.isArray(value)){
      res = [...value];
    }else if(isObject(value)){
      res = {...value};
    }

    // 检测我们浅拷贝的这个对象的属性值有没有是引用数据类型。如果是，则递归拷贝
    Reflect.ownKeys(res).forEach(key=>{
      if(typeof res[key] === "object" && res[key]!== null){
        //此处我们用memo来记录已经被拷贝过的引用地址。以此来解决循环引用的问题
        if(memo[res[key]]){
          res[key] = memo[res[key]];
        }else{
          memo[res[key]] = res[key];
          res[key] = baseClone(res[key])
        }
      }
    })
    return res;  
  }

  return baseClone(value)
}

let studentB = cloneDeep(A)

使用ES插件lodash完成深拷贝

import lodash from 'lodash'

let studentB = lodash.cloneDeep(studentA)

总结

实现深拷贝最简洁的方法是借助JSON的工具方法，使用插件lodash和手写递归算法是适用于实际开发的方法，还请大家根据实际情况自行斟酌。
