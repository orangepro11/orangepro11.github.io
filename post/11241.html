安装

curl https://sh.rustup.rs -sSf | sh

直接回车，安装完成后重启终端。

输入rustup --version

rustup 1.23.1 (3df2264a9 2020-11-30)

输入rustc --version

rustc 1.50.0 (cb75ad5db 2021-02-10)

Hello World!

新建文件hello.rs，用你喜欢的编辑器打开它，我用的vscode。

fn main() {
  println!("Hello World");
}

然后用rustc hello.rs编译它，同级目录下得到同名可执行文件。用./hello.rs即可运行（和gcc类似）。

注意有个感叹号，有同名的println，但是它不是函数，而是一种语言结构（宏规则）。

终端上看到Hello World，得到预期结果。

cargo的使用

创建新项目： cargo new [project name]

在当前目录初始化：cargo init

根据你的情况选择一条命令运行，我运行的是下面的，删除当前目录所有文件，运行命令cargo init，得到一个工作目录。

打开src/main.rs，可以看到它里面为我们写好了Hello World代码。但是这一次我们不需要再用rustc工具去编译它了，直接cargo run就可以了。

可以看到它为我们生成了target目录，并在终端上运行出了结果。

在target/debug 目录下可以找到和你的当前文件夹同名的可执行文件，这个名字可以在Cargo.toml中修改。

编译并运行项目：cargo run

只编译不运行： cargo build

发布release版本：cargo build –release

终端打印

非常非常基础且重要的一个功能，和C语言中的printf一样，rust中的println!有相似的工作逻辑，它只能打印一个格式化的字符串，而不能打印数字，数组，对象之类的东西。所以我们也需要一个类似于占位符的东西，在rust中，它写作一对花括号。

println!("Number:{}",1);

这个占位符是按顺序依次匹配后面传入的参数的。

println!("{} is from {}","orange","amazon");

位置匹配

println!("{0} is from {1} and {0} likes {2}","timelight","china","code");

运行结果：

timelight is from china and timelight likes code

第一个参数的位置编号是0，和大多数语言是一致的（python）。

具名匹配

println!("{name} likes to play {activity}",name="xunanmu",activity="baseball");

运行结果：

xunanmu likes to play baseball

数字格式

以二进制，十六进制，八进制的格式去打印数字10。

println!("Binary: {:b},Hex: {:x}, Octal:{:o}", 10, 10, 10);

运行结果：

Binary: 1010,Hex: a, Octal:12

变量

变量是一门程序设计语言最关键的元素之一，要熟练掌握如何定义和访问一个变量。

新建文件src/vars.rs

pub fn run() {
  let name = "orange";
  let age = "20";
  println!("My name is {} and I'm {} years old",name,age);
}

src/main.rs

mod vars;

fn main() {
    vars::run();
}

注意，一个有点反常识的设计是：let声明的变量是不可被改变的（immutable，它用的词不是const，但是意思是一样的），如果你要改，rust会报一个编译期的错误。如果想声明一个可以更改的变量的话，需要加上mut关键字。

pub fn run() {
  let name = "orange";
  let mut age = "20";
  age = "21";
  println!("My name is {} and I'm {} years old",name,age);
}

这一次编译是能通过的，但是会得到一个警告，意思就是你做的事是多余的，直接把age初始化为21不香吗？

其实rust的设计是合理的，因为大多数定义的变量其实都是常量，如果确实不会更改，声明成常量是对编译器友好的，可以做出更好的优化，只是一直以来它写起来太麻烦所以被人选择性忽略了而已。至于上面的警告，只能说rust为了尽可能的减少bug，在编译期做了太多的工作，致力于让你写出工整高效稳定的代码。

字符串

新建文件src/strings.rs

pub fn run() {
  let hello = "Hello World";
  println!("{}",hello);
}

不过如此，对吗，但是实际上这样是有问题的：这样声明的字符串叫primitive string，它的存储方式是占据固定长度的内存空间，以后不能更改，即字面量，你得到的是它的内存地址，而不是一个实质上的字符串数据类型。如果你希望它存储在heap中以实现可伸长，需要使用String::from()

构造字符串类型

let hello = String::from("Hello World");
println!("{}",hello);

获取字符串长度

let hello = String::from("Hello World");
println!("{}",hello.len());

当然一个中文占三格，这个和其他语言是一样的。

字符串追加

let mut hello = String::from("Hello World");
hello.push('!');
println!("{}",hello);

注意push的是一个字符，如果想push一个字符串，需要用push_str()

获取字符串的容量

很不好理解的概念，实际上这和堆内存空间的申请有关，.len()只是表示当前所占的空间，而capacity是当前段的容量，如果还是不能理解可以看下面的“初始容量”。

let hello = String::from("Hello World");
println!("{}",hello.capacity());

是否为空串

is_empty()，返回boolean值。

是否存在某个字符或子串

let hello = String::from("Hello World");
println!("contains \"World\": {}",hello.contains("World"));

替换子串

let hello = String::from("Hello World");
println!("{}",hello.replace("World","There"));

按空白分割

let hello = String::from("Hello World");
for word in hello.split_whitespace() {
  println!("{}",word);
}

初始容量

let mut s = String::with_capacity(10);
s.push('a');
s.push('b');
println!("{}",s);

字符串是会自动伸长的，但是只有不够的时候才会去申请空间，如果预先给予初始长度就可以让它不用频繁的申请空间，所以最好是按照感觉给它一个大概的初始容量。注意，s的len是2，capacity是10。

断言

当不通过断言的时候会把程序杀死在这一行，并给出错误原因，如果通过了就什么也不会发生。

pub fn run() {
  let mut s = String::with_capacity(10);
  s.push('a');
  s.push('b');

  assert_eq!(2,s.len());

  println!("{}",s);
}

数组

和大多数语言的数组一样，存储相同数据类型的元素。

新建文件 src/arrays.rs

pub fn run() {
  let numbers: [i32;5] = [1,2,3,4,5];
  println!("{:?}",numbers);
  // Get Single Value;
  println!("{}",numbers[0]);
  // Get Array's length
  println!("{}",numbers.len())
  // Get slice
  let slice: &[i32] = &numbers[0..2];
  println!("{:?}",slice)
}

[] 内接受两个参数，第一个是元素的数据类型，第二个是数组要容纳多少个元素，而且第二个参数是必须给出的，rust要求在编译期就知道数组的长度。如果给出的元素类型或者元素数量不匹配，就会给出编译期错误。

如果需要更改数组的元素，仍然要加上关键字mut。转slice的时候可以选择从第几个元素取到第几个元素，上述例子是从第一个元素取到第二个（需要点感觉）。

vector的拷贝

与数组拷贝略有不同，vector的赋值实质上是对内存空间的重新命名，原有名字会丢失对这段内存的引用。也就是如下代码是不能通过编译的。

pub fn run() {
  let vec1 = vec![1,2,3];
  let vec2 = vec1;

  println!("{:?}",(vec1,vec2));
}

报错提示：

2 |   let vec1 = vec![1,2,3];
  |       ---- move occurs because `vec1` has type `Vec<i32>`, which does not implement the `Copy` trait
3 |   let vec2 = vec1;
  |              ---- value moved here
4 | 
5 |   println!("{:?}",(vec1,vec2));
  |                    ^^^^ value used here after move

想要不丢失原有名字对内存的引用，就要。

pub fn run() {
  let vec1 = vec![1,2,3];
  let vec2 = &vec1;

  println!("{:?}",(&vec1,vec2));
}

注意对vec1取值的时候也要用&

结构体

结构体用来自定义数据结构，新建文件src/structs.rs

struct Color {
  red: u8,
  green: u8,
  blue: u8
}

pub fn run() {
  let mut c = Color {
    red: 255,
    green: 0,
    blue: 0
  };
  c.green = 200;
  println!("Color: rgb({},{},{})",c.red,c.green,c.blue);
}

注意结构体的命名要以大写字母开头。

也可以用另一种方式定义结构体。

struct Color(u8,u8,u8);

pub fn run() {
  let mut c = Color(255,255,0);
  c.1 = 200;
  println!("Color: rgb({},{},{})",c.0,c.1,c.2);
}

其实是用tuple的方式定义的结构体。

面向对象？

struct Person {
  first_name: String,
  last_name: String
}

impl Person {
  fn new(first:&str,last:&str) -> Person {
    Person {
      first_name:first.to_string(),
      last_name:last.to_string()
    }
  }

  // Get Full Name
  fn full_name(&self) -> String {
    return format!("{} {}",self.first_name,self.last_name);
  }

  fn set_last_name(&mut self,last:&str) {
    self.last_name = last.to_string();
  }

  // Name to Tuple
  fn to_tuple(self) -> (String,String) {
    return (self.first_name,self.last_name);
  }
}

pub fn run() {
  let mut p = Person::new("Orange", "Juice");
  p.set_last_name("Pro");
  println!("Person: {}",p.full_name());
  println!("Person: {:?}",p.to_tuple());
}

struct用来定义数据结构具有的属性，impl用来定义数据结构的方法，rust不提供new关键字，需要自己实现构造函数的细节。
