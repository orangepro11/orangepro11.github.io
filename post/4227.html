Yarn构建项目

对于不知道响应式编程的概念的人，不用管，直接开始做就可以了，做下去就知道响应式编程的应用场景了。

init

yarn init -y

安装依赖

yarn add rxjs typescript ts-loader webpack webpack-dev-server webpack-cli

调整package.json文件

let observable = Observable.create((observe: any) => {
    observe.next('Hello World');

observable.subscribe((x: any) => {
    console.log(x);
});

直接在浏览器上输出

function print(val: any): void {
    let node = document.createElement('li');
    let textnode = document.createTextNode(val);
    node.appendChild(textnode);
    document.getElementById('output').appendChild(node);
}

然后就可以把console.log()替换成print()了。

尝试在observe中多next一点东西

let observable = Observable.create((observe: any) => {
    observe.next('Hello World');
    observe.next('How are you?');
});

能够看到屏幕上把两个都打印了出来。

complete

let observable = Observable.create((observe: any) => {
    observe.next('Hello World');
    observe.next('How are you?');
    observe.next('多发送一点东西');
    observe.complete();
    observe.next('这个不会被发送');
});

在complete之后的内容不会被发送

处理异常


let observable = Observable.create((observe: any) => {
    try {
        observe.next('Hello World');
        observe.next('How are you?');
        observe.next('多发送一点东西');
        observe.complete();
        observe.next('这个不会被发送');
    } catch (e) {
        observe.error(e);
    }
});

observable.subscribe((x: any) => {
    print(x);
},(error: any) => {
    print(error);
});

在subscribe的第二个函数中监听错误。

已完成状态

observable.subscribe((x: any) => {
    print(x);
},(error: any) => {
    print(error);
},() => {
    print('completed');
});

在第三个参数中处理已完成状态。

有没有感觉和promise的resolve,reject两个状态很像？

所以现在我们知道了它的最常用的应用场景：包裹异步任务

使用定时器的异步操作

let observable = Observable.create((observe: any) => {
    try {
        observe.next('Hello World');
        setInterval(() => {
            observe.next('I am good');
        }, 2000);
    } catch (e) {
        observe.error(e);
    }
});
let observer = observable.subscribe((x: any) => {
    print(x);
}, (error: any) => {
    print(error);
}, () => {
    print('complete');
});

setTimeout(() => {
    observer.unsubscribe();
},6001);

浏览器先打印Hello World然后每隔两秒打印I am good，打印三次后不再打印，异步任务按照我们期待的顺序被执行。

如果有两个观察者呢？

let observer1 = observable.subscribe((x: any) => {
    print(x);
}, (error: any) => {
    print(error);
}, () => {
    print('complete');
});

let observer2 = observable.subscribe((x: any) => {
    print(x);
});

可以看到两个任务几乎是在同时进行，两个观察者可以同时从管道中获取数据。

组合

怎么把他们作为一个整体去操作？

let observer1 = observable.subscribe((x: any) => {
    print(x);
}, (error: any) => {
    print(error);
}, () => {
    print('complete');
});

let observer2 = observable.subscribe((x: any) => {
    print(x);
});

observer1.add(observer2);

setTimeout(() => {
    observer1.unsubscribe();
}, 6001);

两个观察者同时开始同时结束。

和add对应，解除绑定关系，不再演示。
